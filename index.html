📘 Mode d'emploi — travail d'équipe

Cette application ne partage rien automatiquement entre appareils. Chaque personne qui ouvre ce fichier a sa propre copie locale des données, même si le fichier est sur un dossier partagé (OneDrive, etc.). Voici comment travailler correctement en équipe :

1. Un seul poste de saisie officiel. Une personne désignée importe les rondes de terrain (onglet Import de données) et tient le fichier à jour.
2. Partager les données à jour avec l'équipe : après chaque mise à jour, cliquez « Exporter la session (JSON) » ci-dessous et déposez ce fichier sur OneDrive. Les autres membres cliquent « Importer une session » pour voir les mêmes données.
3. Diffuser aux tiers (lecture seule) : cliquez « Générer la version consultation (index.html) » ci-dessous et déposez ce fichier sur votre dépôt GitHub Pages. À refaire après chaque mise à jour.
4. Sauvegarde : le JSON exporté est aussi votre historique/sauvegarde — gardez les anciennes versions si vous voulez pouvoir revenir en arrière.

Structures à monitorer

Un même mandat peut couvrir plusieurs structures distinctes (ex. deux ponts). Chacune a ses propres points, axe, tolérances, rondes et étalon. Un seul rapport PDF est généré à la fois, pour la structure active.

Tolérances et axe PM

Écart entre 2 rondes en-deçà de cette valeur = tendance stable (→).

Points de monitoring (nom canonique)

Ces noms correspondent à la valeur étalon telle que désignée au réseau (ex. 101) — à ne pas confondre avec les codes de mesure du terrain (ex. MON101 ou 101MON), où MON est un simple marqueur ajouté par l'opérateur pour indiquer « valeur mesurée du moment », associé à ce nom à l'étape d'import.

Points de référence (réseau stable)

Points ancrés sur une structure stable, distincts des points de monitoring. Leur stabilité est vérifiée à chaque ronde : un déplacement sur un point de référence remet en question la fiabilité des mesures de la structure surveillée plutôt que de signaler un mouvement réel de celle-ci.

Paires de points (pendage)

Associez un point du bas à son point du haut (ex. 101 / 111) pour calculer automatiquement l'angle de pendage entre les deux, exprimé en angle zénithal (DMS). Utile pour détecter une bascule progressive d'un mur ou d'une structure verticale.

Importer un levé de coordonnées étalon

Format attendu : Point, Nord, Est, Z (PNEZ) ou Point, Est, Nord, Z (PENZ) — le point est le premier élément. Une 5e colonne d'étiquette est optionnelle. Sans étiquette, le nom canonique retenu est le code lui-même. Si une étiquette est présente et commence par REF, le point est classé comme référence ; sinon son préfixe/suffixe MON (marqueur « valeur mesurée ») est retiré pour obtenir le nom canonique.

Pendage des paires de points (angle zénithal)

Angle mesuré depuis la verticale (0° = parfaitement à l'aplomb, 90° = horizontal) entre chaque point du bas et son point du haut associé. Le déversement horizontal indiqué est celui mesuré entre les deux points/prismes eux-mêmes, pas nécessairement celui de la face du mur — les points d'observation ne sont pas nécessairement à égale distance du mur, et il n'est ni pratique ni nécessaire de les y positionner avec cette précision. L'évolution de cet écart d'une ronde à l'autre (comparé à l'étalon) demeure néanmoins représentative d'un éventuel mouvement du mur. « Écart converti » traduit l'écart angulaire en un déplacement linéaire équivalent (mm) à la distance en pente mesurée entre les deux points (distance × tan(écart)) — ex. un écart de 1'30" sur 5 m de distance en pente représente environ 2,2 mm. Configurez les paires dans l'onglet Configuration. Aucune paire configurée = section vide.
